Presentation Företagscase

Den lokala tidningen Nyhetsbladet har sedan 1869 försett den lilla brukskomunen med rykande färska nyheter en gång i veckan. På redaktionen insåg man att omvärlden ändrar sig och att tidningen måste hänga med i svängarna för att klara framtiden. För att komplementera den veckovis utgivna fysiska tidningen, bör man även synas på webben! Till en början med färska nyheter till alla, för att förhoppningsvis följa de större tidningarnas betalmodell senare.

Målgrupp Nyhetsbladet

Redaktionen har som mål att flytta över så många som möjligt till den nya hemsidan, bort från den fysiska produkten. Dock kommer tryckeriet finnas kvar för den målgrupp som absolut inte vill eller kan hänga på trenden. Målgruppen anses enligt kvantitativa studier vara ungefär som de flesta trott, främst människor från kommunen, dock med väldigt bred åldersspann i målgruppen.

Layout

Tidningen har alltid krönts med färgen "ljusblå" som VD Melvin Waltz uttrycker det, därav designas hemsidan efter önskat färgval. då #färgkod redan är ljus försöker designteamet hitta en monokromatisk kontrast mot vit samt andra nyanser av "ljusblå". Tidningens layout eftersträvar normen om en simpel nyhetssida med slider för det mest aktuella nyheterna, exempelvis Kosläppet uppe i Marvik förra veckan myser Waltz. Vidare anses att sidan till en början ska hållas enkel för att minimera utvecklingskostnaden samt komplexitet för den eventuella äldre målgruppen.

Grupparbete

Projektet startade med att ta fram ett case som alla kunde enas om. Vidare valde samtliga gruppmedlemmar en Wireframe från kravunderlaget att arbeta utifrån. Samtliga arbeten genomfördes ekvivalent mot en slutprodukt och inför seminariet så enades gruppen om en produkt som fick genomgå validering och vidare rätta till uppkomna fel. Primära verktyg för arbetet har varit Visual Studio Code, med Git Lens och Go Live extensions samt Github och Github Desktop.

Förbättringar

Gruppens reflektion över förbättringar för Nyhetsbladet är följande:

Javascript

Sidans funktion skulle kunna förbättras avsevärt med mer Javascript. Exempelvis sökformulär och mer avancerad slider.

Användartillgänlighet

Det finns fortsatt förbättringsmöjligheter när det kommer till WCAG 2.1 standarden.

HTML Validering

Valideringen innehåller fortfarande varningar, något som skulle kunna åtgärdas.
